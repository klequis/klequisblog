This tutorial will show you how to create a VirtualBox Virtual Machine snapshot. Things do go wrong and not all of them are easy to correct. Sometimes, the best use of your time is to go back to where you started and try again. Snapshots give you this ability. Before making a significant change to your virtual machine, create a snapshot. If things go wrong you can restore the previous state and try again. Here is how you do it...

Creating a VirtualBox Virtual Machine Snapshot

Launch VirtualBox if it is not already running.

Close your virtual machine if it is running.

Click near the upper right corner.

The screenshot has quite a few snapshots. You may or may not have any. In most cases you will want to choose "Current State".

In the toolbar above the right-hand pane, click the camera button.

Give the snapshot a name.

Click .

In the screenshot 'Before installing Apache (10 seconds ago)' is the 'Current State' is identical for the moment but will not be as soon as you start the virtual machine. 'Before installing Apache' is the name I gave the VirtualBox appends the time '(10 seconds ago)' and always maintains the 'Current State' of the VM.

Working With Snapshots

There are a few things you can do with snapshots.

  • 'Restore Snapshot' will replace the current state with the snapshot - be careful!
  • 'Delete Snapshot' explains itself.
  • 'Show Details' will give you information about the snapshot.
  • 'Clone...'' will make a copy of the VM. You can create a clone with just the current state, with all snapshots or just the currently selected snapshot.

Done! It only takes a minute to make a VirtualBox Virtual Machine snapshot and is well worth you time to do so before each configuration change to your virtual machine.
