This guide will show you how to create a Ubuntu 16.04 Desktop virtual machine. It is part of the series Building a Ubuntu GNOME 16.04 Development Environment. If you are following along with the series you already have Oracle VirtualBox installed and are ready to create your Ubuntu 16.04 (Desktop) virtual machine.

The development environment built in this guide can be used for development with JavaScript, Node.js & PHP. In it you can host Drupal sites, WordPress sites and more.

Ubuntu 16.04 has a desktop and a server version. The desktop version is the correct one for development and general use. The server version has a simple text menu-driven UI not meant for general user interaction.

Requirements

  • VirtualBox installed
  • At least 4GB of RAM
  • At least 20GB of hard drive space
  • 20GB is a conservative number. Your needs will vary. It is best to have more space available.

Definitions

BitTorrent
A peer-to-peer file transfer protocol for sharing large amounts of date over the Internet. When you download via BitTorrent the data comes from multiple sources and is reconstructed into one download.
BitTorrent Client
Software that runs on you computer which is used for downloading via BitTorrent.
Pear to pear
A computer network in which each computer can act as both client and server. In the context of BitTorrent, this means that each computer can be the source a single download to many other computers and can fetch a single download from many other computers. In peer-to-peer, there is not central server.

Is using BitTorrent safe

Historically, there have been some cases where BitTorrent clients have installed malware. My expertise is not computer security. However, I have been using BitTorrent for many years and have not had a problem. As always, you should research software that you are installing from the Internet.

Changing the VM's Settings

At this point my VM is running pretty slow so let's change some settings to get it running faster.

We will be giving more of your host machine's resources to the VM now. This can be tricky. If you give too much of the host's resources to the VM your host will run slow and perhaps be effectively frozen. I have only tried running Ubuntu GNOME on machines with a 4 core processor and 8 GB of RAM. If your host has a lower spec allocate less memory to the VM, then test and increase depending on the performance of your host and guest.

I also recommend closing all applications on the host other than VirtualBox. This won't be necessary once you have things setup, but it may be helpful while you are changing the VM's RAM.

To view the host's resources on Windows bring-up Task Manager. On Ubuntu launch System Monitor.

Do not change the VM's settings while it is running. Doing so can make the VM unusable and difficult to recover.

After all of the above warnings, don't worry. Follow the steps below and all will turn out well.

  1. Shut-down/power-off the VM.
  2. In the VirtualBox Manager, make sure your VM is selected in the left pane.
  3. Click .
  4. Select 'General' on the left and then the ‘Advanced’ tab on the right.
  5. Set ‘Shared Clipboard’ & ‘Drag’n'Drop’ to ‘Bidirectional’. This will make it easier for you to move content between the host and guest.

On an 8 GB system I give 4 or 5 GB to the guest. Look in Task Manager/System Monitor for the amount of free RAM and allocate a portion of it to the VM.

  1. Select 'System' on the left and 'Motherboard' on the right.
  2. Set the amount of RAM in MB per the above recommendation.
  1. Select 'System' on the left and then the 'Processor' tab
  2. If your host computer has 2 cores, give the guest 1. If your host has 4 cores, give the guest 2.

When I am working exclusively on the VM it give the VM 3 of 4 cores.

Again, set a conservative number and then test.

  1. Select 'Display' on the left and then the 'Screen' tab on the right.
  2. Set Video Memory to half or less of the available memory.
  3. Click to save the settings and close the Settings dialog box.
  4. Restart your VM.

Installing Updates

Restart your VM. There are likely some updates ready to be downloaded and installed.

  1. Press Super
  2. Type update
  3. Launch Software Updater and install any updates.
  4. Once the updates are complete, shut down the VM

Ubuntu rarely needs a restart after an update, they don't take long and you can keep working while the updates install. However, this being the first update it may take longer and need a restart. Once updating is done spend some time exploring. ENJOY!

Make a Snapshot

Don't skip this step!

As you are install software on your fresh new VM things can go wrong and not all of them are easy to correct. Creating a snapshot at the end of each major change to your VM will give you a way to go back to a state when things were working so you can try again. Go to this page How to Create a Snapshot With Oracle VirtualBox and make a snapshot now!

A side note on passwords

Since it is running inside of another machine, I don't lock my VM. I lock the host instead. To save some typing I turn-off the VM's screen locking with the below steps.

As I noted above, I use a short & simple password on my VMs. You might be tempted to have no password at all. However, that would change how you used the machine a lot. For example, when you use 'sudo' on the command line there would be no password, nor a password for installing applications. I think that change in the experience of using Linux is a negative for learning.

  1. Log in if you have not already done so.
  2. Press the Super key and type Settings.
  3. Click .
  4. Set Screen Lock to Off.
