I grew-up on Windows. My computing life started less than a year before Windows 3.0 and I did little more than use word processors such as WordStar and WordPerfect. The only command line commands I knew were for listing, moving, renaming and deleting files, and I did very little of that.

Can you be a programmer and not know the command line? I nearly did. From Windows 3.0 to three years ago I developed Windows applications and I only rarely used the comand line. So yes, you can get by with very little command line savy.

But that was then and I think it a bit unusual to not be good with a command line, especially in the Linux world. In fact, I'm not sure that it is possible to completely avoid the command line on Linux.

Using the command line intimidates some people. However, it just takes some getting used to, and it will open-up some possibilities. If you are one of those intimitated by the command line just stick with it. Eventually it will become intuitive

Definitions

For the first 10 characters, the individual characters have the following meaning
- When in the first position it indicates that the item is a file.
When in any of the following 9 positions it means 'the permission is not granted'.
d directory (aka folder)
r Read permission
w Write permission
x Execute permission
l Symbolic link

There are other letters that may appear here but these are the ones you will see most often.

It is confusing to have a user named 'karl' and a group named 'karl' in the example. I'll try to clear that up right after the next table. If after that it is causing confusion, leave a comment and I'll find a better approach.

Explanation of Output Parts

user, group, other
RefPartExplanation
A-"-" (hyphen) indicates it is a file and not a directory which would be "d". There are other designators that can exist in this position. I have listed them in a table below.
Brw-The first grouping of three (characters 2-4) is the permissions for the user. In this case user karl has read & write but not execute.
Crw-The second grouping of three (characters 5-7) is the permissions for the group. In this case, same as previous, the group karl has read & write but not execute.
Dr--The third grouping is the permissions for everyone else. In this case every other than user karl and members of the group karl have read but not write and not execute.

That is confusing but I will clear it up some in a minute. -rw-rw-r-- There are 10 characters in this string The first caracter will be "-" for a file and "d" for a directory "-" means it is a file, not a directory "d" means it is a file Then there are 3 groupings of 3 characters. The 3 characters in each group are r, w x (rwx) which mean "r" means read permission "w" means write permission "x" means execute permission "-" means the permission is not granted (does not have the permission) The first group of 3 is the owner's permissions The second group of 3 is the owning group's permissions The third group of 3 are the permissions for everyone else * They are always in the same order Move / rename Make directory Delete file Delete directory Copy List permissions with number Change permission on files onlyList directory Change permissions Change owner (user) Change owner (group) View beginning of file View end of file View a file Search within a file
