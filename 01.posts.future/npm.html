About NPM

NPM is not only for using other people's modules/packages. You can use it for your own project to modulaize you project and make it possible for multiple developers to integrate their work via npm modules.

It isn't just for server side node.js, there are modules that are not for node.js such as packages that enhance the commendline and front-end package such as bootstrap..

Features of interest

  • Organizations

Important Topics

  • Installing npm packages locally
  • Use a package.json file
  • The semantic versioner for npm $npm install semver
  • installing globally (for things like grunt where you want to use it from the command line)
  • npm deduping $ npm dedupe
  • npm3 Non-determinism. Depending on install order the tree in node_modules may differ between machines. However, all the dependencies are there and there is not actual difference to the application. To get the tree structures to match, delete the node_modules folder and run $ npm install.

NPM Cheatsheet

check node version | $ node -v
install latest version of node | $ npm@latest -g
find path to node install directory | $ npm config get prefix
install a module | npm install
see if you have outdated packages (means there are updates that meet your semver rules. It wont change anything) | npm outdated
get updates to packages currently in project | $ npm update
uninstall a local package | $ npm uninstall --save || --save-dev
list outdated global packages | npm outdated -g --depth=0
update global packages | npm install -g
add - update global packages for npm ver > 2.6.1 | npm update -g
uninstall global package | npm uninstall -g
see the npm tree in terminal | $ tree -d node_modules/ OR $ npm ls
see top level / primary dependencies | npm ls --depth=0
get rid of duplicate modules in dependency tree | $ npm dedupe

granular semver ranges

Semver for consumers

If you were starting with a package 1.0.4, this is how you would specify the ranges:
* Patch releases: 1.0 or 1.0.x or ~1.0.4
* Minor releases: 1 or 1.x or ^1.0.4
* Major releases: * or x
