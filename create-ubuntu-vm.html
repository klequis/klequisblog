This guide will show you how to create a Ubuntu 16.04 Desktop virtual machine. It is part of the series Building a Ubuntu GNOME 16.04 Development Environment. If you are following along with the series you already have Oracle VirtualBox installed and are ready to create your Ubuntu 16.04 (Desktop) virtual machine.

The development environment built in this guide can be used for development with JavaScript, Node.js & PHP. In it you can host Drupal sites, WordPress sites and more.

Ubuntu 16.04 has a desktop and a server version. The desktop version is the correct one for development and general use. The server version has a simple menu-driven UI which is not very user friendly.

Requirements

  • VirtualBox installed
  • At least 4GB of RAM
  • At least 20GB of hard drive space
  • 20GB is a conservative number. Your needs will vary. It is best to have more space available.

Changing the VM's Settings

At this point my VM is running pretty slow so let's change some settings to get it running faster.

  1. Shut-down/Power-off the VM
  2. In the VirtualBox Manager, make sure your VM is selected in the left pane.
  3. Click
  4. Select the ‘Advanced’ tab
  5. Set ‘Shared Clipboard’ & ‘Drag’nDrop’ to ‘Bidirectional’. This will make it easier for you to move content between the host and guest.

For the next step you need to know how much memory your host computer has. If you don't know, look it up now.

I have not tested using the VM on a host with less than 8 GB of RAM. I typically give 3 or 4 GB to the guest. With 4 - 5 GB remaining minus what the host OS uses, the host is left with 2 to 3 GB free. If you host is Windows or iOS, my guess is the host needs at least 1 GB to itself. Start out low and work your way up based on the host and guest performance. To help others out, please leave you memory/RAM configurations in the comments section.

  1. Set the amount of RAM in MB per the above recommendation.
  1. Select 'System' on the left and then the 'Processor' tab
  2. If your host computer has 2 cores, give the guest 1. If your host has 4 cores, give the guest 2.

    3. When I am working exclusively on the VM it I give it 3 of 4 cores

  1. Select 'Display' on the left and then the 'Screen' tab on the right.
  2. Set Video Memory to half or less of the available memory
  3. Click to save the settings and close the Settings dialog box.
  4. Restart you VM

Installing Updates

There are likely some updates ready to be downloaded and installed

  1. Press Super
  2. Type update
  3. Launch Software Updater and install any updates.
  4. Once the updates are complete, shut down the VM

Make a snapshot

Don't skip this step!! Things do go wrong and not all of them are easy to correct. Creating a snapshot at the end of each major change to you VM will give you a way to go back to a state when things were working so you can try again. Go to this page How to Create a Snapshot With Oracle VirtualBox and make a snapshot now!.
