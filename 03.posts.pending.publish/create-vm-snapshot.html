Things do go wrong and not all of them are easy to correct. Sometimes, the best use of your time is to go back to where you started and try again. Snaps give you this ability. Before making a significant change to your virtual machine, create a snapshot. If things go wrong you can restore the previous state and try again. Here is how you do it...

Creating a Snapshot

  1. Launch Virtualbox if it is not already running.
  2. Close your virtual machine if it is running.
  1. Click near the upper right corner.
  1. In the toolbar above the right-hand pane, click the camera button.
  1. Give the snapshot a name.
  2. Click .

Done!

In the picture 'Before installing Apache (10 seconds ago)' is the snapshot. 'Current State' is identical for the moment but will not be as soon as you start the virtual machine. 'Before installing Apache' is the name I gave the snapshot. VirtualBox appends the time '(10 seconds ago)' and always maintains the 'Current State' of the VM.

Working With Snapshots

There are a few things you can do with snapshots
  • Restore Snapshot will replace the current state with the snapshot - be careful!
  • Delete Snapshot explains itself.
  • Show Detais will give you information about the snapshot.
  • Clone... will make a copy of the snapshot. Clone is useful when you want to preserve the Current State.
