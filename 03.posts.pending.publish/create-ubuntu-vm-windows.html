This guide will show you how to create a Ubuntu 16.04 Desktop virtual machine. It is part of the series Building a Ubuntu 16.04 Development Environment. If you are following along with the series you already have Oracle VirtualBox installed and are ready to create your Ubuntu 16.04 (Desktop) virtual machine. The development environment built in this guide can be used for development with JavaScript, Node.js & PHP. In it you can host Drupal sites, WordPress sites and more.

Ubuntu 16.04 has a desktop and a server version. The desktop version is the correct one for development and general use. The server version has a simple menu-driven UI which is not very user friendly.

Requirements

  • VirtualBox installed
  • At least 4GB of RAM
  • At least 20GB of hard drive space
  • 20GB is a conservative number. Your needs will vary. It is best to have more space available.

There is often a problem at this point. The machine will shut-down, begin to restart but then get stuck. There are various tips on discussion forums for how to fix this but I find the following works.
  • First, take a 5 minute break. The machine may start on its own.
  • If waiting does work then, select Machine>Reset. This should successfully restart the machine.

At this point I usually unlock all the icons from the Launcher (vertical bar like Windows taskbar on left) except for Files and Firefox. The others are rarely used. If you want the Guest Additions CD icon to go away, right-click on it and select Eject.

Changing the Virtual Machine's Settings

Now we are going to change the settings on the virtual machine from the VirtualBox Manager. If your virtual machine is running, shut it down before beginning this step. Editing a virtual machines settings while it is running can cause problems.
  1. Make sure the virtual machine is shut-down.
  2. Right-click on the name of your virtual machine the left pane and choose 'Settings...
    1. Select the 'Advanced' tab
    2. Set 'Shared Clipboard' & 'Drag'nDrop' to 'Bidirectional'. This will make it easier for you to move content between the host and guest.

For the next step you need to know how much memory your host computer has. If you don't know, look it up now.

I have not tested using the virtual machine with less than 3 GB of RAM. Remember, we are going to install a web server, database server and run several more applications at the same time. So while it may run now with approximately 750 MB, you will need more for the final development environment. My host has 8 GB and I usually set the guest to 4 GB. With Windows using some memory and the guest taking 8 GB, I have 2 GB left on the host for running applications, which I do, but I keep it to a minimum while the virtual machine is running. So I don't have a solid/tested recommendation for machines with less memory. My guess is that you should leave at least 1 GB of RAM for the host. Try differing and increasing amounts and see what works for you.

    1. Set the amount of RAM in MB per the above recommendation.
    1. Select 'System' on the left and then the 'Processor' tab
    2. If your host computer has 2 cores, give the guest 1. If your host has 4 cores, give the guest 2.
The more video memory you give the guest the snappier its UI will be. Start out by giving the guest about half of the video memory. You can change this number later according to how the host and guest are performing.
    1. Select 'Display' on the left and then the 'Screen' tab on th right.
    2. Set Video Memory to half or less of the available memory
    3. Click to save the settings and close the Settings dialog box.

You can also select up to 8 monitors. I have 2 monitors and this feature works very well.

  Congratulations again! Your Ubuntu virtual machine is installed and configured. Start it up and check it out.

Make a snapshot

Don't skip this step!! Things do go wrong and not all of them are easy to correct. Creating a snapshot at the end of each major change will give you a way to go back to a state when things were working. See How to Create a Snapshot With Oracle VirtualBox.
