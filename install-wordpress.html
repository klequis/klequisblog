This guide will show you how to install WordPress on your GNOME Ubuntu development machine.

Do not put WordPress configured in ths way on a machine which is accessibl from the internet as it would be compromised quickly. This configuration is only for a development machine which cannot be reached from the intenet.

Create a Database User

If you already have a database user you want to use you can skip this step. If you only have root then follow the steps below to create a new user.

Don't use root. Sure, on this database server which is isolated by VirtualBox and likely your router it should not be a problem. However, it is such a bad practice that it should never be done.

Log into phpMyAdmin using root

Click on the 'User accounts' tab.

pic.001.useraccounts.png

Click on 'Add user account'.

pic.002.click.adduser.png

Fill in the 'User name'. I use my name: 'karl' and a simple 4 character password. Again, if this were a production database I would use a less obvious user name and a strong password.

pic.003.login.info.png

Leave both options under 'Database for user account' unchecked.

Under 'Global privileges' mark 'Check all' selected.

At the bottom of the page click .

Test the New User

To test the new user, log out of phpMyAdmin and then log back in with the new user.

It is important to test the new users. If 10 steps from now something doesn't work you won't know if it is what you just did or if your user is not setup properly. Test everything you can as soon as you can.

Create database > Click New in the left pane > Fill in the 'Database name' and click . pic.006.new.database.png > You will see the new database in the left pane. Download Wordpress Go to https://wordpress.org/download and click on the blue 'Download WordPress x.x.x' button in the top right of the page. pic.007.dwnld.wp.png. Extract

If you are new to setting-up websites it might be best to use the same folder name as I will below. This can help make configuration easier. I will be referring to the folder by the name I use.

> Right-click on the downloaded file and choose 'Extract Here'. This will create a 'wordpress directory' pic.008.extract.wp.png. > Move the extracted 'wordpress' directory to ~/workspace/ > Rename ~/workspace/wordpress to ~/workspace/wptutorial or a name of your choosing. Setup wp-config.php > Inside the /wptutorial directory find the file 'wp-config-sample.php' and make a copy of it. > Rename the copied file 'wp-config.php'. > Open wp-config.php in gedit. > Change DB_NAME to the name you have your new database. > Change DB_USER to the name of your new user. > Change DB_PASSWORD to the password for your new user. > Note that DB_HOST is already set to 'localhost'. pic.009.wp-config.defaults.png pic.010.wp-config.new.values.png > Save and close the file. Create a Virtual Host for Wordpress

Detailed instructions on how to create a virtual host can be found here. cd /etc/apache2/sites-available sudo cp phpinfopage.conf wptutorial.conf sudo gedit wptutorial.conf sudo gedit /etc/hosts sudo a2ensite wptutorial.conf sudo systemctl restart apache2 open http://wptutorial.localhost/wp-admin/install.php Select your language and click . pic.001.select.language.png Fill-in the requested information. The Install Wordpress button is not enabled until you enter a strong password or 'Confirm use of weak password'. Click .pic.012.welcome.information.png Click . 013.success.png Enter your username and password, then click . pic.014.login.png That's it! WordPress is installed.
