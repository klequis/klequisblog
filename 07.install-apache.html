Apache is a free web server. According to W3Tech', Apache is the second most widely used web server at 33.7%. Third place goes to Microsoft IIS with 9.2%.

This guide tells you how to install Apache on Ubuntu 16.04 for use as a development web server

For this tutorial we will be using the command line from Terminal. Each command that you need to enter will start with '$'. Type the command exactly as shown and then press enter. If prompted with something like 'continue (y/n)' enter 'y' and press enter. Enter the commands one at a time.

All of the commands will start with 'sudo'. This gives the command the ability to make system wide changes using superuser priviledges (aka 'root') so that the command has permission to make all necessary changes.

WARNING

The below instructions are for setting-up Apache for use on a development machine that is not deployed to or exposed to the internet. If you want to install Apache on a machine that is exposed to the internet see Apache HTTP SERVER PROJECT.

Installing Apache

OK, open Terminal and let's get started.

$ sudo apt install apache2

Test you installation with this command.

$ sudo systemctl status apache2

Press q to exit the status screen

As an additional test we will launch the 'Apache2 Ubuntu Default Page'. Open your web browser and navigate to http://localhost. If installation was successful you will see a page as shown.

If Apache didn't pass the above tests, use these commands and then try opening localhost in the browser again.

$ sudo systemctl enable apache2
$ sudo systemctl start apache2

Apache Configuration

Since this is a development mahcine that will not be accessible from the internet, we can make a few changes to the configuration that will make it easier to work with. Do not make this instance of Apache accessbile over the internet as it is not properly secured.

Open the 'envvars' configuration file.

$ sudo i- gedit /etc/apache2/envvars

To make Apache run under your account, change the user on the below two lines. The picture shows the lines edited to have my user name.

export APACHE_RUN_USER=yourUserName
export APACHE_RUN_GROUP=yourUserName

Finally, we need to enable the Rewrite module.

$ sudo a2enmod rewrite
$ sudo systemctl restart apache2
