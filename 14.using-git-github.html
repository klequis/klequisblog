Using Git with GitHub

This guide will show you how to install Git and then use Git with GitHub. This is not a complete Git tutorial but is a very good start for someone who has not used it before. There is a very good online book ffor using Git. If you are new to Git, reading the first three chapters will be well worth the time. See the See Also section at the bottom of the post.

There are different ways you can create a repository and work with it. The process below is one possible process and one that I think it is easy for a new Git/GitHub user to follow and learn.

To check the status of your repository execute the below command.

$ git status

Link many specialized subjects, Git has its own vocabulary and understanding Git well enough to make use of it requires that you learn it. Here is what some of the words & phrases in the 'git status' results mean:

master
Git is linke a tree. You can create branches to you tree. The default branch is called 'master'. Look at your GitHub home page and find the dropdown that says "Branch: master"
origin
'orign' is a name has associated with the repository on your local machine. As we will see later, it holds the URL to the repository on GitHub.
'nothing to commit'
In a semple sense this means, you have nothing ready to make permenant in the repository. In Git language it says you have nothing in the repository's staging area
'working directory is clean'
This means there is nothing in your project's directory that is not already in the Git repository on you local machine.

Note that the status message only tells you about the state of the repository on you local machine, it says nothing about the repository on GitHub.

Here are some definitions to help with the next step. These are 'plain-language' definitions, not official Git definitions.

Untracked
An untracked file has never been added to Git.
Committed
A file that has been put into Git using $ git commit
Staged
A staged file is ready to be added to Git. It is in the 'staging area' but has not been 'committed' yet. A file is staged with the $ git add command. You can use $ git add on a new file or a perviously committed file that has been since been changed.
