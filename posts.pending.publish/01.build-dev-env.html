There are many ways to build a development environment and the developer's preference and needs will differ. For a developer in a large corporation having a web server running on his/her local machine is often not allowed, whereas a freelancer will need a local web server and a lot more. The following post is how I create a development environment that works for me. It may not be suitable for you.


A good development environment can make you more productive. A poor one will rob you of your sleep. klequis - from experience

JavaScript, Node.js, WordPress, Drupal & probably other things that I don't do.

Important Notes

  1. This guide is for a development machine running inside a virtual box that will not accept web request except from localhost. It installs Apache under the users local account which gives it privileges that a web server exposed to the internet should not have. If you are like me, your production site is hosted and the web server will be setup and secured by the hosting company. You will develop your site on your local machine and move the final/updated site to the hosted/production environment. Given this arrangement, running Apache under your local account is safe.
  2. The below process will allow you to build your sites in your home directory (/home/workspace) which is more accessible than the default which is /var/www/html.
  3. Don't ignore errors! After executing a command, scan the resulting output for error messages. If any are found, research the solution and ensure the problem is fixed before proceeding to the next step.
  4. Make snapshots: One advantage to a virtual machine is that you can save its current configuration/state in snapshots. If you make a change to the machine that doesn't work out and you can't correct it, you can simply go back to one of your snapshots and the change will be undone.
  5. If you are not accustom to a Unix/Linux environment remember - everything is case sensitive.
  6. All steps that begin with '$' are to be typed into terminal.
  7. The instructions are largely a pictorial. Match your installation UI to the picture. Be sure to read any text above a picture before attempting that step. Some of it is crucial.
