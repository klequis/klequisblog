A little over a year ago I got back into development and wanted a free editor. I tried Sublime, Brackets and Atom. Brackets wouldn't install on my Linux machine and although there is likely a way to get it working I felt an editor that needed research to install was a bad bet. I very much liked Sublime but didn't want to deal with the nagging to pay for it. It is well worth paying for but I wasn't up to it at that time. As a result I switched to Atom and found it easy to configure compared to Sublime and so I have stuck with it since.

Out of the box Atom is a very simple editor. However, there are, at this moment, 6,023 packages available to customize atom in terms of functionality and appearance. To find the packages you want you can search the atom repository by the type of functionality you are looking for and/or Goggle for Atom packages of interest.

Atom is often criticized for being slow and unable to open very large files. Neither of these present a problem for me. Atom does take some number of seconds to launch. However, I don't open and close it much during the day so it isn't a problem for me. Atom has crashed when I was opening a very large file, say bootstrap.css, but I rarely do that and have other editors on hand for when I do such as gEdit and Sublime.

Below we will install Atom, install packages and then change a few settings in the editor. The setup is based on how I have Atom setup. My focus is the JavaScript, HTML, CSS & React. If you are working with these technologies my setup will be a good starting point for you.

Are you a new programmer that hasn't settled on an editor yet? Choose one of the most popular ones and stick with it. Knowing your editor well, particularly the keyboard shortcuts, configuration and addons, will improve you productivity.

Installing Atom

If you do not have Atom installed go to atom.io and download the install for your system. The installation instructions can be found in the Atom Flight Manual.

Installing Packages

You can install packages using the Atom UI but the fastest way is to use the $ apm command from the command line. The format is

$ apm install package1 package2 packageN

Here is the command to install all the packages listed above

apm install highlight-selected seti-ui pigments atom-beautify auto-detect-indentation rest-client git-plus merge-conflicts autocomplete-modules custom-title expose todo-show git-time-machine goto-definition atom-ternjs autocomplete-paths docblockr glowing-cursor markdown-preview-plus linter linter-csslint linter-eslint linter-htmlhint open-in-browser browser-plus language-babel

Here are the editor settings I prefer

In Atom choose Edit>Preferences and then the Editor tab on the left.

Change settings to match the below.

  1. Scroll Past End = checked
  2. Show Indent Guide = checked
  3. Show Invisibles = checked
  4. Soft Tabs = checked
  5. Soft Wrap = checked
  6. Tab Length = 2

A nice theme with icons which show file type

The Seti theme was installed when we use the 'apm' command above. It needs to be activated.

  1. In the settings page, choose 'Themes' on the left
  2. Set UI Theme = Seti

I set the syntax theme to Base16 Tomorrow Dark
