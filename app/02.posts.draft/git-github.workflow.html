Collaborating with GitHub

If you don't have an organization to work in, create one. Complete the following steps in the organization.

01. Click 'New' to start creating a new repository.

02. Fill in a name anc click Create repository

03. Create the project on your local machine

04. Create a local git repo and commit it

05. Execute these two lines

06. !Delete the entire project from your local machine. Now you are ready to start as if you were working in a shared repo.

In GitHub, go to the workflow home page

07. Click Clone or download and then the 'Copy to clipboard' icon

08. Extcute these commands

$ git clone https://github.com/trivalleycoders-org/workflow.git $ cd workflow (missing 009)

09. Git creates a remote named 'origin' for you. It is best to rename it to avoide confusion that may occur when multiple contributors have a remote called origin.

$ git remote -v $ git remote rename origin klequis $ git remote -v

10. Your current branch is 'master'. Make a branch to make your changes in. You name plus a suffix works well.

$ git branch -av $ git branch klequis-workflow $ git branch -av

11. Checkout the branch

12. Make some changes to one of the project files and save it. Git status will show a modified file.

13. Add and commit your changes

14. Push the changes to GitHub

15. Now take a look at your new branch on the GitHub page

16. Create a pull request

Click on the 'Compare & pull request' button

17. Fill in a title and comment, then click 'Create pull request'

You are done.

Merging a pull request into master

You must have agreement from the team to perform the next step.

18. Got to the 'Pull requests' tab

19. Click on 'Merge pull request'

20. Fill in the details and then click 'Confirm merge'

21. The changes you made in the branch are now in master
