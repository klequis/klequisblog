This guide will show you how to install WordPress on your GNOME Ubuntu development machine.

This post is for WordPress on a machine that is not accessible to the internet. Do not put WordPress configured as outlined below on a machine that is accessible from the internet as it will be quickly compromised. In this series we are installing WordPress on a virtul machine inside of VirtualBox. By default, VirtualBox is configured such that the Apache web server on the VM/guest can not be access except from within the VM. This means that the WordPress site being installed can be accessed from a browser running on the VM/guest, but cannot be accessed from the internet or even from the host machine.

Create a Database User

If you already have a database user you want to use you can skip this step. If you only have root then follow the steps below to create a new user.

Don't use root. Sure, on this database server which is isolated by VirtualBox and likely your router it, should not be a problem. However, it is such a bad practice that it should never be done.

Log into phpMyAdmin using root.

Click on the 'User accounts' tab.

If you already have a database user setup you can use it and skip this step.

Click on 'Add user account'.

Fill in the 'User name'. I use my name: 'karl'.

Set 'Host name' to 'Local'

Enter a password. I use a simple 4 character password. If this were a production database I would use a less obvious user name and a strong password.

Leave both options under 'Database for user account' unchecked.

Under 'Global privileges' mark 'Check all' selected.

At the bottom of the page click .

Test the New User

To test the new user, log out of phpMyAdmin and then log back in with the new user.

It is important to test the new users. If 10 steps from now something doesn't work you won't know if it is what you just did or if your user is not setup properly. Test everything you can as soon as you can.

Create database

Click New in the left pane.

Fill in the 'Database name' and click .

You will see the new database in the left pane.

pic.006.new.database.png

Download WordPress, Extract It and Move to /Workspace

Go to https://wordpress.org/download and click on the blue 'Download WordPress x.x.x' button in the top right of the page body. (Don't get tricked-out by the similar button under the search box. It seems to do nothing.)

pic.007.dwnld.wp.png

If you are new to setting-up websites it might be best to use the same folder name as I will below. This can help make following the configuration steps easier. I will be referring to the folder by the name I use.

Right-click on the downloaded file and choose 'Extract Here'. This will create a 'wordpress' directory.

Move the extracted 'wordpress' directory to ~/workspace/

Rename ~/workspace/wordpress to ~/workspace/wptutorial or a name of your choosing.

pic.008.extract.wp.png

Edit wp-config.php

Inside the /wptutorial directory find the file 'wp-config-sample.php' and make a copy of it.

Rename the copied file 'wp-config.php'.

Open wp-config.php in gedit.

Change DB_NAME to the name of your new database.

Change DB_USER to the name of your new user.

Change DB_PASSWORD to the password for your new user.

Note that DB_HOST is already set to 'localhost'. Leave it as is.

Save and close the file.

Create a Virtual Host for Wordpress

Detailed instructions on how to create a virtual host can be found here.

Copy phpinfopage.conf to wptutorial.cong

$ cd /etc/apache2/sites-available
$ sudo cp phpinfopage.conf wptutorial.conf

Open wptutorial.conf and change each instance of 'phpinfopage' to 'wptutorial'. The file should be exactly as shown in the picture.

$ sudo gedit wptutorial.conf
pic.010.01.wptutorial.conf.png

Open /etc/hosts and make an entry for the new virtual host.

$ sudo gedit /etc/hosts

Enter the line below.

127.0.0.1    wptutorial.localhost

Save and close the hosts file.

pic.010.02.hosts.png

Enable the new host and restart Apache.

$ sudo a2ensite wptutorial.conf
$ sudo systemctl restart apache2

You will see in terminal a message that instructs you to use 'service apache2 reload' to restart Apache. This is the older command that still works. I suggest you use the newer command in the event the older one gets phased-out.

Run the WordPress Installation Script

Open http://wptutorial.localhost/wp-admin/install.php

Select your language and click .

pic.011.select.language.png

Fill-in the requested information. The Install WordPress button is not enabled until you enter a strong password or 'Confirm use of weak password'.

Click .

Click .

pic.013.success.png

Enter your username and password, then click .

pic.014.login.png

Congratulations - WordPress is installed!
