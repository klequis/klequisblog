Introduction

For those new to Git and GitHub it can appear complicated and the workflow may seem long-winded. It isn't either of those things. Like any specialized tool, it has its own vocabulary, which to the new user is like a small foreign language. Trust me, Git and GitHub will make your life easer and your code safe from disaster. Once you learn it you will wonder how you lived without.

This tutorial series will teach you the basics of Git and GitHub with examples and exersizes to increase your understaning. It will start with a very simple workflow that is adequate for someone working on their own or building a small project. It will then move into more advance but still basic features that are used daily by developers. When you complete the tutorial you will be able to effectively use Git and GitHub to manage your code.

This tutorial makes heavy use of the command line for manulipating files. If you are not comfortable with the command line please go through the tutorial Using the Command Line & Understanding Permissions. Alternatively, you could perform the file operations using a file manager.

All the Git commands shown will work in Linux, Mac and Windows. However, the file manipulation commands will not work on Windows. Windows user may want to use a file manager or functionality in their editor for creating files and folders.
