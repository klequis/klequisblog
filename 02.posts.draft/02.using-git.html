Using Git

Creating a Sample Project

It doesn't matter to Git how you create your project files. You can create them in a file manager, code editor or via the command line (which I'll show you below). All that matters is that all project files are located under one master folder commonly referred to as 'root'. In our case, the root directory will be named 'gittutorial'.

Every developer has their set of conventions for organizing their projects. One of mine is to put all projects in a folder named 'workspace'. Let's follow that convention for the tutorial. In Terminal, create a folder named workspace/ and make it the current directory.

$ mkdir workspace
$ cd workspace

The $ touch command is used to create a file or to change the files date/time. For more information see this page.

... and then execute the following commands. Don't worry what they mean for now, they will be explained below.

$ mkdir gittutorial
$ cd gittutorial
$ touch index.html README.md
$ mkdir css
$ cd css
$ touch style.css

Your project structure should now look like this:

Let's put some content in the files.

index.html 
<!DOCTYPE html>
<head>
  <title>Git Tutorial</title>
  <link rel="stylesheet" href="css/style.css">
</head>
<body>
  <h1>Hello World, as usual.</h1>
</body>
style.css 
body {
  background-color: lightgray;
}
README.md 
#Git Tutorial ReadMe

Open index.html in a browser.

Using Git to Manage Project Files

We will be using the following commands.

  1. $ git init
  2. $ git status
  3. $ git add
  4. $ git commit

$ git init - Create a New Git Repository

$ git init creates a new empty git repository. It needs to be executed from your project root, so let's get there now.

$ cd ~/workspace/gittutorial

Now execute ...

$ git init

... and then look in your file manager to see the '.git' folder that contains the reposigory. This is a hidden folder. If your file manager is not showing hidden files & folders, you will need to change a setting.

$ git status - Show the Status of the Repository

To see the status of your repository use the below command.

$ git status

Note where it says 'Untracked files'. These are files that are in the project folder but not yet in the Git repository.

$ git add - 'Stage' (Prepare) Files to be Committed

Adding files is a two step process. Remember, full explanations of what is happening are given later in the tutorial. Just go with the flow for now.

$ git add --all
$ git status

Note that Git status shows 'Changes to be committed' and the files are labeled 'new file'. The files are not yet in the repository. There is one last step.

$ git commit - Add Files to the Git Repository

$ git commit -m "initial commit"
$ git status

The -m means 'message'. The output of $ git status is a bit crypitc but you can see both your files have been added with 'create mode'. The directory structure has been preserved as well.

Part of the $ git status message says 'nothing to commit, working directory clean'. This indicates that all files and changes are in the Git repository.

Adding Changes to the Git Repository

So far we have added new files to the repository. Adding changes changes that have been made to those files follows the same process. Let's do that now. Make the changes shown in bold.

index.html 
<!DOCTYPE html>
<head>
  <title>Git Tutorial</title>
  <link rel="stylesheet" href="css/style.css">
</head>
<body>
  <h1>Hello World, as usual.</h1>
  <p>Adding file changes to Git</p>
</body>
style.css 
body {
  background-color: burlywood;
}

Using $ git status now shows two modified files.

$ git status

Next 'add' the changes and note that 'status' shows the files as 'modified' instead of 'new file'.

$ git add --all
$ git status

Now commit the changes.

$ git commit -m "changed body color and added a paragraph"
$ git status

Status once again shows 'working directory is clean' which means all files and changes are in the repository.

Summary

Doing all that with some explanation and comments makes it seem like a long process. It really isn't. Here is what we did with Git.

$ git init // is only done once
$ git add
$ git commit -m "initial commit"
$ git add
$ git commit "changed body color and added a paragraph"

The next part of this tutorial will show you how to push your Git repositiry to GitHub.

Fantastic! You are now using Git to manage your project.
